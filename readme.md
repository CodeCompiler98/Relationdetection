# Relationship Classifier Algorithm

This project takes a JSON file pulled off GDELT and extracts relevant groups, relationship edges, and values to build a graph representation, where every node is a group and every edge is a (or a group of) documentated relationships between the groups. The goal is to use the sentiment of each relationship edge to connect groups without direct edges and logically label the relationship as positive(1), negative(0) or neutral(-1) based on the path between the two groups. Our final output has the lable, a path trace and a confidence score.

## Project Overview
- Graph construction from `bangladesh_news.json` with aggregated edge attributes (frequency, goldstein, strength, cost, etc.).
- Community detection (Louvain) used for optional confidence adjustments.
- Pathfinding via Dijkstra on `cost` to find actor-to-actor paths.
- Path logic: sign-product with log1p-strength accumulation and length decay; outputs `sign` and `confidence`.

## Repository Structure
### Main Scripts
- `graphbuild.py` — builds the graph based on the JSON and computes its stored values (called by network_analysis)
- `network_analysis.py` — main logic, takes the graph and runs Dikstras algorithm, louvain algorithms, and path/confidence logic to provide the user results
### Benchmark Scripts
- `relation_csv.py` — Builds a CSV that holds every single possible node pair for benchmarking purposes based on the graph 
- `accuracy_benchmark.py` — evaluate relationship algorithm on hand-labeled samples
- `time_benchmark.py` — runtime scaling experiments and plotting
- `bangladesh_news.json` — dataset used to build the graph

## Quickstart

Install dependencies:

```bash
pip install -r requirements.txt
```
Run the algorithm on a pairing of your choice (or run benchmark shown below, those call the same algorithm)
```bash
python network_analysis.py
```

Run the full CSV export for benchamrking/testing all possible values (writes `node_relations.csv`):

```bash
python relation_csv.py
```

Run the scaling benchmark and save plot:

```bash
python time_benchmark.py
```

Run evaluation on manual benchmark (prints accuracy with optional confidence filtering):

```bash
python accuracy_benchmark.py
```

## Outputs
- `node_relations.csv` — contains results of a full graph benchmark: actor1, actor2, relation_exist, relation_label, confidence, runtime_seconds
- `relation_scaling.png` — plot generated by time_benchmark.py
- `network.png` - shows the fully build graph and its edges generated by graph_build.py 

## Testing
- Use `accuracy_benchmark.py` to reproduce confidence and accuracy numbers based on the hand-labeled CSV benchmark.
